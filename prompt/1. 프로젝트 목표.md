## 기능 구현

- authToken
  - 로그인 여부를 확인 한다.
  - localStorage에 저장 된다.
  - 구성 요소 (JSON)
    - server : {server type}
    - site : {site name}
    - service : {service option}
    - wpayUserKey : {wpayUserKey}
    - wtid : {wtid}
    - userId : {userId}
    - createdAt : {현재 시간(yyyy-mm-dd hh24:mi:ss)}
    - updatedAt : {최초 세팅은 createdAt과 동일}
    - expiresAt : {updatedAt을 기준으로 6 시간 이후 시간(yyyy-mm-dd hh24:mi:ss)}
      - expiresAt 값 갱신
        - updatedAt값이 변경 되면 expiresAt값을 갱신 한다.
    - uu : {6자리 10진수 랜덤 생성 값.}
      - uu 값 갱신
        - updatedAt값이 변경 되면 uu값을 갱신 한다.
    - sgn : {wpayUserKey + wtid + userId + createdAt + updatedAt + expiresAt + uu를 shift연산한 값 + hashKey를 문자열로 연결하여 sha256으로 암호화한 값}
      - uu Shfit 연산 방법.
        - 첫 번째 자리: 홀수 = Shfit 왼쪽 / 짝수 = Shfit 오른쪽
        - 두 번째 자리: Shift 연산 이동할 bit 수
        - 세 번째부터 6번째 자리: Shift 연산 수행.
      - hashKey는 secrets.ts의 HASH_KEY를 사용 한다.
      - sgn 값 갱신
        - authToken 요소 값이 변경 되면 sgn값을 갱신 한다.

- 로그인 기능 구현
- 메인 페이지 기능 구현
  - 로그인 상태 확인
    - localStorage에서 authToken 값 확인.
    - authToken이 없으면 로그인 페이지로 이동
    - authToken이 있으면 유효성을 검증 한다.
      - expiresAt이 현재 시간보다 이전이면 authToken이 유효하지 않음.
      - sgn이 유효하지 않으면 authToken이 유효하지 않음.
