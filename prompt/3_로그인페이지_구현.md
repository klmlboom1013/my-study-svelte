# 로그인 페이지 구현

## UI 구성

- UI Style
  - 2_UI_STYLE.md 참고

- 정렬
  - 페이지 중앙에 위치 한다.

- 서비스 선택
  - Dropdown Box & text Input 기능 혼합.
    - Dropdown Box 로 원하는 서비스는 선택하거나 텍스트를 입력하여 찾는것이 가능 하다.
    - values
      - wpaystd-old, wpaystd2, wpaypro, wpayplus, wpaycst
    - default value : wpaystd2

- 서버 선택
  - Radio Button 기능
    - DEV, STG, PROD 중 하나만 선택이 가능 하다.
    - default value : DEV
    - PROD 선택 시 추가 선택
      - Modal 로 어떤 PROD 서버로 접속할 건지 선택해야 한다.
      - 선택은 Radio Button 기능으로 제공 한다.
        - GLB Domain, KS Domain, FC Domain
        - 세로로 정렬 한다.
      - default value : GLB Domain

- 로그인 사이트 선택
  - Dropdown Box & text Input 기능 혼합.
    - Dropdown Box 로 원하는 사이트는 선택하거나 텍스트를 입력하여 찾는것이 가능 하다.
  - default value : 빈값으로 하며 "선택해 주세요" 메시지를 표시 한다.

- 로그인 사이트의 채널 선택
  - Dropdown Box & text Input 기능 혼합.
    - Dropdown Box 로 원하는 채널은 선택하거나 텍스트를 입력하여 찾는것이 가능 하다.
  - default value : 빈값으로 하며 "선택해 주세요" 메시지를 표시 한다.

- 로그인 ID 입력
  - text Input
  - hint : wpayTestUser01
  - focus in 시 hint 가 사라진다.
  - focus out 시 입력된 값이 없으면 hint 가 나타난다.

- 휴대폰 번호 입력
  - number Input
    - 복사한 문자열을 붙여넣기 할 경우 숫자를 제외한 문자를 제거한다.

- 페이지에서 세팅한 데이터 저장 기능
  - 체크박스 : remember me
    - 체크박스를 체크하면 로컬 스토리지에 페이지에서 세팅한 데이터를 저장한다.
    - 체크박스를 체크하지 않으면 로컬 스토리지에 페이지에서 세팅한 데이터를 저장하지 않는다.  

- 로그인 버튼
  - 서비스, 서버, 로그인 사이트, 로그인 사이트의 채널, 로그인 ID, 휴대폰 번호를 기반으로 로그인을 한다.
  - 활성화: 서비스, 서버, 로그인 사이트, 로그인 사이트의 채널 값이 모두 선택 되었을 때
    - Click 시 메인화면으로 이동 한다.
      - 로그인 ID가 없으면 Default 값으로 wpayTestUser01으로 로그인 한다.
  - 비활성화: 서비스, 서버, 로그인 사이트, 로그인 사이트의 채널 값이 모두 선택 되지 않았을 때
  - 버튼 문구
    - 활성화: 로그인
    - 비활성화: 로그인 (disabled)
