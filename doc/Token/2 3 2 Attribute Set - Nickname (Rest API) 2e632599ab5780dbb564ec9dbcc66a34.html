2.3.2. Attribute Set - Nickname (Rest API)

WpayToken 별칭 설정 (Rest API)

  • URI: HTTPS /stdwpay/std/rest/v1/setncnm
  • HTTP Method: POST (Rest / content-type: application/www-from-urldecoded-x)
  • 서비스 설명: WPAY에 이미 등록된 신용카드 또는 계좌 결제정보에 사용자가 구분하기 쉬운 별칭을 부여합니다.

1. Request (Rest)

가맹점 서버에서 이니시스로 별칭 설정을 요청하는 전문입니다.

NameTypeReqLenEncDescURLsign
midStringO10가맹점 ID1
wpayUserKeyStringO100OWPAY 사용자 KEY [CI 1:1 매칭]2
wpayTokenStringO100O별칭을 설정할 결제수단 WPAY Token3

상세 설명 및 구현 인사이트

1. 사용자 편의성 제고: 사용자가 여러 장의 카드나 계좌를 등록한 경우, 마스킹된 번호만으로는 구분이 어려울 수 있습니다. 이 API를 통해 별칭을 설정하면 결제 수단 선택 시 사용자가 본인의 수단을 더 쉽게 식별할 수 있습니다.

2. 데이터 무결성: signature 생성 시 wpayTokenNcnm (별칭 값)이 포함되어야 합니다. 이는 별칭 데이터 자체가 전송 과정에서 위변조되지 않았음을 보장하기 위함입니다.

3. 암호화 준수: 사용자의 고유 식별 값인 wpayUserKey와 결제 수단 식별 값인 wpayToken은 반드시 **암호화(Enc: O)**된 상태로 전송되어야 보안 규격을 충족합니다.

4. 연계 활용: 설정된 별칭은 결제수단 정보 조회 또는 결제정보 리스트 조회 시 wpayTokenNcnm 필드를 통해 다시 받아볼 수 있으며, 이를 가맹점의 결제 수단 선택 화면에 그대로 노출할 수 있습니다.

비유를 통한 이해: 결제정보 별칭 설정은 마치 여러 권의 똑같은 통장(결제수단)에 각각 '생활비', '적금', '비상금'이라고 견출지(별칭)를 붙이는 것과 같습니다. 겉모양(카드사, 마스킹 번호)이 비슷해서 헷갈리는 통장들에 이름을 붙여줌으로써, 나중에 돈을 쓸 때(결제할 때) 실수 없이 원하는 통장을 바로 꺼낼 수 있게 돕는 기능입니다.
