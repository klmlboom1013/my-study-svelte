2.3.1. Attribute Set - Primary (Rest API)

WpayToken 주 결제수단 설정 및 해지 (Rest API)

  • URI:

◦ 주 결제수단 설정: HTTPS /stdwpay/std/rest/v1/setmaj

◦ 주 결제수단 해지: HTTPS /stdwpay/std/rest/v1/rlsmaj

  • HTTP Method: POST (Rest / Content-Type: application/www-from-urldecoded-x)
  • 서비스 설명: 가맹점에서 등록된 결제정보 중 특정 항목을 주 결제수단으로 설정하거나 해지 요청을 처리합니다.

상세 안내 및 연동 인사이트

1. 독립적 관리 체계: WPAY는 신용카드군과 계좌군에 대해 주 결제수단을 각각 별도로 관리합니다. 즉, 주 카드가 설정되어 있어도 주 계좌를 따로 설정할 수 있습니다.

2. 보안 적용: wpayUserKey와 wpayToken은 사용자 식별 및 결제 권한을 가진 핵심 정보이므로 반드시 **암호화(Enc: O)**하여 전송해야 합니다.

3. Signature 검증: 데이터 위변조 방지를 위해 mid, wpayUserKey, wpayToken, ci 값을 순서대로 조합한 뒤 가맹점의 hashKey를 더해 생성한 signature를 포함해야 합니다.

4. 사용자 경험(UX) 활용: 이 API를 통해 주 결제수단이 설정되면, 가맹점은 이후 결제 리스트 조회 시 majYn 필드가 'Y'인 항목을 확인하여 결제 화면 상단에 우선적으로 노출하거나 기본 선택 상태로 제공할 수 있습니다.

비유를 통한 이해: 주 결제수단 설정은 마치 지갑에서 가장 자주 쓰는 카드를 맨 앞 주머니에 꽂아두는 것과 같습니다. 결제할 때마다 지갑을 뒤져 카드를 고를 필요 없이, 이 API를 통해 '가장 먼저 사용할 카드'를 지정해 둠으로써 결제 과정을 훨씬 빠르고 간편하게 만드는 역할을 합니다.
