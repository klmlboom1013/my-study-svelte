5.2. Mypage (Web UI)

WPAY 회원설정 – 마이페이지 (Web UI)

  • URI: HTTPS /stdwpay/std/u/v1/mypage
  • HTTP Method: POST
  • 서비스 설명: 결제수단 관리(등록/삭제), 결제 비밀번호 변경, 서비스 해지 등의 기능을 제공하는 WPAY 마이페이지 화면을 호출합니다.

1. Request (Form Submit)

가맹점에서 이니시스 마이페이지 호출 시 전송해야 하는 파라미터입니다.

NameTypeReqLenEncDescURL
midStringO10가맹점 ID
wpayUserKeyStringO100OWPAY 사용자 KEY [CI 1:1 매칭]
returnUrlStringO256서비스해지 처리 결과를 받을 Callback URLO
signatureStringO100데이터 위변조 검증 값
arsVersionString1ARS인증방법 [1:생년월일 6자리, 2:난수]
authNoLengthString1arsVersion=2일 때 난수 길이 [2, 3, 4 중 선택]

Request Signature 생성 Plain Text 순서: mid={value}&wpayUserKey={value}&returnUrl={value}&hashKey={발급받은 hashKey value}

2. Response (Callback returnUrl / POST)

사용자가 마이페이지 내에서 서비스 해지를 완료한 경우, 설정된 returnUrl로 결과 데이터가 전송됩니다.

NameTypeLenEncDesc

Response Signature 생성 Plain Text 순서: resultCode={value}&resultMsg={value}&mid={value}&wtid={value}&wpayUserKey={value}&hashKey={발급받은 hashKey value}

주요 특징 및 연동 가이드

1. 통합 관리 기능: 마이페이지 하나로 결제수단 등록/삭제, 비밀번호 변경, 서비스 해지 기능을 한 번에 제공하므로 가맹점이 각 기능을 개별적으로 구현할 필요가 없어 개발 효율성이 높습니다.

2. 보안 사항: wpayUserKey는 민감 정보이므로 반드시 암호화(Enc: O) 처리가 되어야 하며, 위변조 방지를 위해 signature 값을 정확한 순서에 따라 생성해야 합니다.

3. UI 커스터마이징: 가맹점의 브랜드 아이덴티티를 유지할 수 있도록 resetCssUrl이나 customTitle 등을 통해 화면의 디자인과 타이틀을 맞춤 설정할 수 있습니다.

4. 해지 결과 처리: 마이페이지 내의 다른 활동(비밀번호 변경 등)은 해당 화면 내에서 종료되지만, 서비스 해지가 발생하면 가맹점의 returnUrl로 데이터가 포스트(POST)되므로 가맹점은 이를 받아 자사 DB의 회원 상태를 업데이트해야 합니다.

비유를 통한 이해: WPAY 마이페이지는 마치 **아파트의 '관리사무소'**와 같습니다. 입주민(사용자)이 비밀번호를 바꾸거나, 출입 카드(결제수단)를 새로 등록하고 반납하며, 심지어 이사(서비스 해지) 절차까지 한 곳에서 모두 처리할 수 있는 통합 창구 역할을 하기 때문입니다. 가맹점은 이 관리사무소로 가는 통로만 잘 연결해 두면 됩니다.
