4.3.2. Search List (Rest API)

정기결제 신청 리스트 조회

  • URI: HTTPS /stdwpay/std/rest/v1/billinginfos
  • HTTP Method: GET
  • 서비스 설명: 가맹점에서 WPAY 정기결제 신청 리스트 조회를 요청합니다.

1. Request (Rest / URL Parameters)

조회 조건으로 status, wtid, wpayToken 등을 선택적으로 사용할 수 있습니다.

상세 안내 및 연동 인사이트

1. 조회 조건의 활용: 특정 결제 수단에 대한 정기결제 상태만 보고 싶다면 wpayToken을, 특정 신청 건을 확인하려면 신청 당시의 wtid를 파라미터에 넣어 호출하면 정밀한 조회가 가능합니다.

2. 보안 준수: wpayUserKey와 wpayToken은 사용자 식별 및 결제에 직결되는 민감 정보이므로 반드시 암호화(Enc: O) 처리가 필요하며, GET 방식이므로 URL Encoding을 잊지 말아야 합니다.

3. 정기결제 관리: 응답으로 받은 billingList 내의 **status**를 통해 가맹점은 사용자가 현재 어떤 정기결제 서비스를 유지하고 있는지(00) 또는 해지했는지(02)를 파악하여 관리자 페이지 등에 노출할 수 있습니다.

4. WTID와 Token의 역할: billingList 안의 wtid와 wpayToken은 실제 자동결제 승인 요청을 할 때 핵심 파라미터로 사용되므로 가맹점 데이터베이스와 대조하여 무결성을 확인하는 용도로 활용하기 좋습니다.

비유를 통한 이해: 이 API는 **"구독 서비스 관리 장부를 열어보는 것"**과 같습니다. 사용자가 우리 가게에서 어떤 상품들을 정기 배송(자동결제) 시켜두었는지, 그중에서 지금 배달이 가고 있는 것은 무엇인지(00), 배달을 끊은 것은 무엇인지(02) 한눈에 목록으로 뽑아 확인하는 절차입니다.
