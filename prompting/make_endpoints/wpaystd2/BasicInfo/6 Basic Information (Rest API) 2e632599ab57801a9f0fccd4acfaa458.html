6. Basic Information (Rest API)

WPAY 가맹점정보 통합 조회

  • URI: HTTPS /stdwpay/std/rest/v1/getBasicInfos
  • HTTP Method: GET
  • 서비스 설명: 가맹점의 기준 정보(할부, 포인트, 할인 등)를 통합 조회하며, 데이터 암호화 및 signature 생성 후 URL 파라미터 세팅 시 URL Encoding이 필요합니다.

1. Request (Rest / URL Parameters)

NameTypeReqLenEncDescURLsign
midStringO10가맹점 ID1

optionsString20기준정보 조회 옵션 (빈값: All)

조회 옵션(options) 상세: 한 개 이상의 옵션을 조합할 경우 구분자 **":"**를 사용합니다 (예: MQ:CP:CE).

  • MQ: 최대 할부개월 / CP: 카드포인트 / CE: 즉시할인
  • NI: 무이자할부 / PK: 결제수단정보 / UC: 사용가능 카드정보
  • BI: BC카드 즉시할인 대상 발급사 정보

Request Signature 생성 Plain Text 순서: mid={value}&hashKey={발급받은 hashKey value}

상세 안내 및 연동 인사이트

1. 효율적인 정보 관리: 가맹점은 결제 페이지를 구성하기 전 이 API를 호출하여 현재 사용 가능한 카드사나 진행 중인 무이자 이벤트 정보를 실시간으로 받아와 사용자에게 보여줄 수 있습니다.

2. 보안 절차: GET 방식 호출이므로 암호화된 데이터와 signature를 URL 파라미터로 전달할 때 반드시 URL Encoding을 수행하여 특수문자로 인한 데이터 유실을 방지해야 합니다.

3. 선택적 조회: 모든 정보가 필요하지 않은 경우 options 파라미터를 활용해 필요한 항목(예: 즉시할인 CE만 조회)만 골라 응답 데이터의 크기를 줄이고 통신 효율을 높일 수 있습니다.

4. WTID 활용: 응답으로 제공되는 wtid는 해당 조회 트랜잭션을 식별하는 고유 번호로 활용됩니다.

비유를 통한 이해: 가맹점정보 통합 조회는 마치 식당에 들어가기 전 입구에 붙어 있는 '오늘의 메뉴 및 할인 안내판'을 확인하는 것과 같습니다. 손님이 주문(결제)을 하기 전에, 지금 이 식당에서 어떤 카드로 할인이 되는지, 몇 개월까지 무이자가 되는지 미리 정보를 파악하여 최선의 결제 방식을 선택할 수 있도록 돕는 역할을 합니다.

WPAY 가맹점정보 통합 조회(Rest API) 응답(Response) 전문의 상세 구조 중, 질문하신 **nointInfoList.nointInfos**와 discList.discInfos 리스트 내부의 필드 구성을 소스 내용을 바탕으로 상세히 설명해 드립니다.

이 두 리스트는 가맹점이 사용자에게 제공할 수 있는 무이자 할부 혜택과 즉시 할인 혜택의 세부 조건을 담고 있는 핵심 데이터입니다.

1. 무이자 정보 리스트 (nointInfoList.nointInfos)

가맹점에서 특정 카드사별로 제공하는 무이자 할부의 세부 조건(금액 및 개월 수)을 포함하는 리스트입니다.

필드명 (Name)타입 (Type)길이 (Len)상세 설명 (Description)
amountString3무이자할부 적용 가능 금액<br>단위는 만 원이며, 10,000 * amount로 계산됩니다. 예를 들어 값이 '5'라면 5만 원 이상 결제 시 적용됨을 의미합니다.
monthString2무이자할부 개월 리스트<br>무이자가 적용되는 할부 개월 수 정보를 제공합니다.
  • 연동 팁: nointInfoList는 상위에 bankCardCode(카드사 코드)와 cardName(카드 브랜드명)을 가지고 있어, 가맹점은 이를 통해 "OO카드로 X만 원 이상 결제 시 Y개월 무이자"와 같은 문구를 화면에 구성할 수 있습니다.

카드사별로 적용되는 즉시 할인 이벤트의 구체적인 금액과 적용 조건을 포함하는 리스트입니다.

필드명 (Name)타입 (Type)길이 (Len)상세 설명 (Description)
discAmtString10즉시할인 금액<br>실제 결제 시 차감되는 할인 금액입니다.
minAmtString10즉시할인 적용 최소금액<br>할인을 받기 위해 필요한 최소 주문 금액입니다.
maxAmtString10즉시할인 적용 최대금액<br>할인이 적용되는 최대 주문 금액 한도입니다.

정리 및 유의사항

1. 데이터 구조: basicInfos 객체 내에 nointInfoList와 discList가 존재하며, 그 안에 다시 각 카드사별 세부 정보를 담은 nointInfos와 discInfos가 리스트 형태로 중첩되어 있습니다.

2. 연쇄 활용: discList.discInfos에서 조회된 **couponCode**는 신용카드 결제 인증(payauth/card) 요청 시 필수 파라미터로 사용될 수 있으므로 별도로 관리하는 것이 좋습니다.

3. 옵션 활용: 만약 무이자 정보만 필요하거나 즉시할인 정보만 필요한 경우, 요청 시 options 파라미터에 NI(무이자) 혹은 CE(즉시할인) 등을 세팅하여 필요한 리스트만 선별적으로 받아볼 수 있습니다.

이 정보들을 적절히 조합하면 사용자가 결제 수단을 선택하는 시점에 가장 유리한 혜택(무이자 또는 할인)을 실시간으로 안내할 수 있어 결제 전환율을 높이는 데 도움이 됩니다.
